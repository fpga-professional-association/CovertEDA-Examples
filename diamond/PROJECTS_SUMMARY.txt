PROJECT 1: BLINKY LED (Beginner)
================================================================================

Purpose: Simple LED blinker with rotating pattern
Target Device: LFE5U-25F-6BG256C (ECP5, 1,280 slices)
Clock: 25 MHz oscillator

Verilog Modules:
  - blinky_top.v (135 lines)
    * 25MHz -> 1Hz prescaler (25M divider)
    * 4-bit LED shift register for rotating pattern
    * 28-bit counter for LED sequencing

Constraints:
  - blinky.lpf
    * Clock: P3 (LVCMOS33, 25MHz)
    * Reset: T14 (LVCMOS33 with pull-up)
    * LED outputs: B2, C2, A2, B1 (12mA LVCMOS33 drivers)
    * Bank A & B configured for 3.3V I/O

Reports:
  - timing.twr: 287.5 MHz Fmax (11.5x margin), 36.5ns slack
  - utilization.mrp: 34 LUTs, 24 FFs, 0.7% device usage
  - power.rpt: 22.3mW total (18.1mW dynamic, 4.2mW static)
  - drc.rpt: All checks PASSED, proper decoupling recommendations
  - pad.rpt: Pin assignment verification, bank voltage compatibility

Key Features:
  ✓ Fully synthesizable prescaler design
  ✓ Asynchronous reset distribution
  ✓ Power-down unused I/O banks
  ✓ Realistic thermal analysis (45.3°C junction @ 25°C ambient)

PROJECT 2: UART BRIDGE (Intermediate)
================================================================================

Purpose: USB-UART bridge with dual FIFOs for data buffering
Target Device: LFE5U-45F-8BG381C (ECP5, 2,304 slices)
Clock: 48 MHz

Verilog Modules:
  - uart_top.v (120 lines)
    * Master interface to USB simulator
    * FIFO controller for RX/TX buffering
    * Activity-based LED status (3 LEDs)
    * Configurable baud rate (4-bit select)

  - uart_core.v (180 lines)
    * 8-bit UART with 8N1 configuration
    * Configurable baud rate: 9600 to 115200
    * Independent TX and RX state machines
    * Prescaler-based timing (up to 5208 divider @ 48MHz)

  - fifo_sync.v (95 lines)
    * Parameterizable synchronous FIFO
    * Configurable WIDTH and DEPTH (used: 8-bit x 512)
    * Separate read/write clocks (same domain)
    * Full/empty/count flags

Constraints:
  - uart.lpf
    * Clock: J4 (48MHz)
    * UART TX/RX: N14, M14
    * USB interface: 8 data pins + control signals (24 pins)
    * Status LEDs: T20, T19, T18
    * All LVCMOS33, pull-ups on inputs

Reports:
  - timing.twr: 156.0 MHz Fmax, 7.5ns slack (marginal RX path)
  - utilization.mrp: 412 LUTs, 298 FFs, 4 EBR blocks (1.9%)
  - power.rpt: 35.6mW total, 52.1°C junction temperature

Key Features:
  ✓ Full UART implementation (TX/RX independent)
  ✓ Dual 512-byte FIFOs (4 EBR blocks)
  ✓ Activity monitoring with LED indicators
  ✓ Realistic baud rate calculations
  ✓ Synchronizer for metastability prevention

PROJECT 3: SERDES LOOPBACK (Advanced)
================================================================================

Purpose: High-speed serial loopback with PRBS pattern testing
Target Device: LFE5UM5G-85F-8BG756C (ECP5 with SERDES, 7,680 slices)
Clock: 312.5 MHz reference (2.5 Gbps SERDES operation)

Verilog Modules:
  - serdes_top.v (115 lines)
    * PLL instantiation for clock multiplication
    * SERDES TX/RX wrapper instances
    * PRBS generator/checker control
    * Status LED mapping (8 LEDs for diagnostics)

  - serdes_wrapper.v (145 lines)
    * 8:1 serializer (TX) and 1:8 deserializer (RX)
    * Differential I/O simulation (LVDS)
    * Parallel-to-serial conversion logic
    * 32-bit data path (4 bytes parallel)

  - prbs_gen.v (65 lines)
    * PRBS7 polynomial: x^7 + x^6 + 1
    * Taps at positions [7, 6, 1]
    * Parameterizable output width (8, 16, 32-bit)
    * Non-zero initialization (0x01)

  - prbs_check.v (90 lines)
    * PRBS7 pattern verification
    * Bit-by-bit error detection
    * Adaptive lock detection with threshold
    * 8-bit error counter with saturation

Constraints:
  - serdes.lpf
    * Reference clock: G23 (312.5MHz, LVCMOS33)
    * SERDES TX pair: C28/D28 (LVDS25 differential)
    * SERDES RX pair: A28/B28 (LVDS25 differential)
    * Status LEDs: A14-A7 (8 pins for diagnostics)
    * Differential pair constraints (50ps skew, 100ps jitter limits)

Reports:
  - timing.twr: 312.5 MHz (1.0x margin - at speed), 1.044ns slack (tight)
  - utilization.mrp: 1245 LUTs, 890 FFs, 2 SERDES blocks, 1 PLL
  - power.rpt: 156.7mW total, 67.2°C junction, SERDES dominates (59.6%)

Key Features:
  ✓ Full SERDES instantiation for 2.5 Gbps operation
  ✓ PLL for clock multiplication (312.5MHz * 8 = 2.5GHz internal)
  ✓ PRBS7 test pattern generation and verification
  ✓ Marginal timing at speed (requires excellent PCB design)
  ✓ Realistic LVDS differential pair routing
  ✓ Per-rail power analysis for high-speed circuits

PROJECT 4: WISHBONE SOC (Intermediate-Advanced)
================================================================================

Purpose: Simple Wishbone bus SoC with SRAM, GPIO, and Timer slaves
Target Device: LFE5U-85F-6BG554I (ECP5, 6,912 slices)
Clock: 50 MHz

Verilog Modules:
  - soc_top.v (135 lines)
    * CPU simulator with test pattern generation
    * Wishbone master generating sequential accesses
    * Status LED outputs (4 LEDs)
    * Debug serial TX output

  - wb_intercon.v (85 lines)
    * Address decoding (3 address spaces)
    * Slave multiplexing (address ranges: 0x000, 0x100, 0x200)
    * Data and ACK aggregation
    * Combinational address/control logic

  - wb_sram.v (65 lines)
    * 1024 x 32-bit = 4 KB dual-port SRAM
    * Byte-select write support (4 independent write enables)
    * Synchronous read/write
    * Initialized with test pattern

  - wb_gpio.v (120 lines)
    * 4-bit input with synchronization
    * 4-bit output register
    * Input change detection for interrupts
    * Programmable interrupt mask
    * 4 independent control registers

  - wb_timer.v (125 lines)
    * 32-bit counter with 16-bit prescaler
    * Configurable limit register for wraparound
    * Interrupt generation on limit match
    * Programmable prescale and enable

Constraints:
  - soc.lpf
    * Clock: P4 (50MHz, LVCMOS33)
    * GPIO input: V20-T20 (4 pins)
    * GPIO output: V19-T19 (4 pins)
    * Status LEDs: R20, R19, R18, R17
    * Debug UART: Q20 (TX), Q19 (RX)
    * Wishbone bus timing constraints (WB_SETUP 10ns, WB_HOLD 2ns)

Reports:
  - timing.twr: 125.0 MHz Fmax, 11.998ns slack (2.5x margin)
  - utilization.mrp: 3456 LUTs, 2134 FFs, 16 EBR blocks (7.4%)
  - power.rpt: 78.9mW total, 48.7°C junction (very safe)

Key Features:
  ✓ Full Wishbone B3 compliant implementation
  ✓ Single master (CPU simulator), 3 slaves
  ✓ Address-based slave selection (12-bit address space per slave)
  ✓ 4KB SRAM with byte-select writes
  ✓ Interrupt-capable GPIO and Timer
  ✓ Excellent timing margin (2.5x) for reliable operation
  ✓ Low power design (78.9mW with significant headroom)

PROJECT 5: VIDEO SCALER (Advanced)
================================================================================

Purpose: High-speed video scaler from 720p to 1080p
Target Device: LFE5U-85F-8BG756C (ECP5, 7,680 slices)
Clock: 148.5 MHz pixel clock (1080p@60Hz)

Verilog Modules:
  - scaler_top.v (100 lines)
    * Video pipeline controller
    * Line buffer interface
    * Interpolator control
    * Sync signal passthrough
    * 8-bit debug status output

  - line_buffer.v (50 lines)
    * 1280 x 24-bit dual-port RAM
    * Simultaneous read/write capability
    * Synchronous operation at 148.5 MHz
    * Uses 2 EBR blocks (36 Kbits per block)

  - interpolator.v (110 lines)
    * Linear vertical interpolation for 1.5x scaling
    * 32-bit intermediate calculations
    * State machine for interpolation phase control
    * 8-bit interpolation weight (0-255)

  - pixel_counter.v (60 lines)
    * Pixel and line timing generator
    * 1920x1080 @ 60Hz video format
    * Horizontal counter: 0-2199 (H_TOTAL = 2200)
    * Vertical counter: 0-1124 (V_TOTAL = 1125)
    * Line/frame end signals

Constraints:
  - scaler.lpf
    * Pixel clock: J2 (148.5MHz, LVCMOS33)
    * Video input: 24 LVDS25 differential pairs for pixel data
    * Sync inputs: VSYNC, HSYNC, VALID (LVCMOS33)
    * Video output: 24 LVDS25 differential pairs
    * Sync outputs: VSYNC, HSYNC, VALID
    * Debug status: 8 LVCMOS33 pins
    * Constraints: VIDEO_PIXEL_SKEW 50ps, VIDEO_SYNC_JITTER 100ps

Reports:
  - timing.twr: 165.3 MHz Fmax, 0.687ns slack (MARGINAL - at limit)
  - utilization.mrp: 5678 LUTs, 4123 FFs, 32 EBR, 8 DSP blocks
  - power.rpt: 234.5mW total, 72.4°C junction (thermal margin: 27.6°C)

Key Features:
  ✓ High-speed video pipeline (148.5 MHz pixel clock)
  ✓ LVDS differential pairs for video signal integrity
  ✓ Dual line buffering for interpolation support
  ✓ 8 DSP blocks used for RGB multiplication (interpolation)
  ✓ Tight timing at device speed limit (marginal 0.687ns slack)
  ✓ Realistic power dissipation for video processing
  ✓ Thermal management considerations included

REPORT FORMATS (All Realistic Lattice Diamond Output)
================================================================================

1. timing.twr (Timing Report)
   - Constraint summary table with period/slack analysis
   - Clock domain analysis with rising/falling edge definitions
   - Setup/hold slack by path group
   - Critical path identification and detailed analysis
   - Maximum frequency estimation
   - Timing constraint verification
   - Pin-level timing summary
   - Total violations/warnings count

2. utilization.mrp (Map Report)
   - Design statistics (slices, LUTs, FFs, EBR, DSP, PLL)
   - Detailed resource allocation
   - Connectivity analysis (routing tracks, net distribution)
   - Power analysis summary
   - Performance metrics
   - Device utilization percentage
   - Compile time statistics
   - Optimization recommendations

3. power.rpt (Power Calculator Report)
   - Total power breakdown (dynamic vs. static)
   - Per-rail analysis (VCORE, VCCIO, VCCAUX, etc.)
   - Component power attribution (logic, memory, I/O, clock, DSP)
   - Functional block power analysis
   - Thermal analysis (junction temperature, θJA, margin to max)
   - Frequency and switching activity
   - Operating scenarios (idle, partial, full activity)
   - Optimization recommendations
   - Operating condition variations (temp, voltage, process)

4. drc.rpt (Design Rule Check) - blinky_led only
   - 15+ individual rule checks
   - I/O configuration verification
   - Clock timing validation
   - Reset distribution analysis
   - Voltage compatibility matrix
   - PLL and SERDES checks
   - Memory and DSP utilization
   - Bank saturation analysis
   - Global routing resource verification
   - Timing rule verification
   - Hardware PCB recommendations

5. pad.rpt (Pad Report) - blinky_led only
   - Pin assignment summary
   - Bank configuration details
   - Voltage standard specifications
   - Drive strength and slew rate
   - Power pin distribution
   - Decoupling capacitor recommendations
   - Electrical characteristics
   - Manufacturing notes and PCB layout guidance
